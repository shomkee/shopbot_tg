# 🛍 Бот для продажи цифровых товаров

Телеграм-бот на **Python (aiogram 3)** с админ-панелью, категориями, подкатегориями, платёжными системами **xRocket** и **CryptoBot**, реферальной системой и промокодами.

## ✨ Возможности

1. Категории с отдельным просмотром
2. Оплата через **xRocket** и **CryptoBot**
3. Подкатегории (неограниченная вложенность)
4. Профиль: ID, сумма покупок, сумма депозитов, история покупок и выгрузка покупки файлом
5. Реферальная система (ссылка в профиле)
6. Промокоды (создаёт только владелец)
7. Админка: товары, промокоды, выдача баланса, блокировка
8. Кнопка поддержки (аккаунт + правила)
9. Пополнение баланса
10. Покупка через категории → товар
11. Описание товара + фото (меню с фото и inline-кнопками)
12. Основные кнопки — под клавиатурой, остальные — inline

## 🚀 Запуск

```bash
# 1. Установите зависимости
pip install -r requirements.txt

# 2. Скопируйте конфиг и заполните его
cp .env.example .env
nano .env   # впишите BOT_TOKEN и OWNER_ID

# 3. Запустите бота
python bot.py
```

## ⚙️ Настройка

Все настройки — в файле `.env` (см. `.env.example`).

- **BOT_TOKEN** — токен от [@BotFather](https://t.me/BotFather)
- **OWNER_ID** — ваш Telegram ID (узнать у [@userinfobot](https://t.me/userinfobot))
- **CRYPTOBOT_TOKEN** — токен Crypto Pay API
- **XROCKET_TOKEN** — ключ xRocket Pay API

```
├─ keyboards.py        # клавиатуры

│  ├─ shop.py          # каталог и покупки
│  ├─ profile.py       # профиль, история, рефералы, промокоды
│  ├─ payments.py      # пополнение баланса
│  └─ admin.py         # админ-панель
└─ providers/          # интеграции платежей
   ├─ cryptobot.py
   └─ xrocket.py
```

## 📝 Как пользоваться админкой

1. Откройте бота → кнопка **▲ Панель** (видна только админам).
2. Создайте категорию → при желании подкатегорию.
3. Добавьте товар (название, описание, цена, фото по желанию).
4. Добавьте сток: каждая строка = одна единица товара (аккаунт/ключ/текст).

При покупке бот автоматически спишет баланс и выдаст одну единицу товара из стока.

## ⚠️ Важно о платежах

Проверка оплаты реализована по кнопке «Проверить оплату» (polling), без webhook — это проще в запуске. Эндпоинты API xRocket/CryptoBot могут меняться — см. `providers/cryptobot.py` и `providers/xrocket.py`, там всё легко подправить под актуальную документацию.
